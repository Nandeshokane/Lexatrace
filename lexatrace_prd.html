Six progressive build phases. Each phase has clear deliverables and is self-contained enough to demo independently.
01
Project Setup & Auth Foundation
Weeks 1–2 · Backend scaffolding, auth, CI/CD
  • Week 1Init monorepo: backend/ (FastAPI) + frontend/ (React + Vite). Setup PostgreSQL + Docker Compose.
  • Week 1Configure Alembic for DB migrations. Create base User, Project, and AnalysisJob models.
  • Week 2Implement JWT-based auth: register, login, refresh token. Hash passwords with bcrypt.
  • Week 2Build React auth screens (login / signup). Setup React Router, Zustand for global state, Axios interceptors for token refresh.
  • Week 2Setup GitHub Actions CI: lint (ruff + eslint), test runner (pytest + vitest), Docker build check on every PR.
02
File Ingestion & Storage Pipeline
Weeks 3–4 · Multi-format upload, preprocessing, queue
  • Week 3Build FastAPI file-upload endpoint. Accept PDF, DOCX, TXT, PNG/JPG, ZIP (source code). Validate MIME types server-side.
  • Week 3Integrate MinIO (S3-compatible) for file storage. Store raw files with a UUID key + metadata in PostgreSQL.
  • Week 3Build text extractors: pdfminer.six for PDFs, python-docx for DOCX, pygments for source code tokenization.
  • Week 4Setup Celery + Redis task queue. On file upload, dispatch async preprocessing task. Store job status in DB.
  • Week 4Build React upload page: drag-and-drop zone, file type indicator, real-time job progress via WebSocket or polling.
03
NLP & Semantic Similarity Engine
Weeks 5–8 · BERT embeddings, similarity scoring, copyright detection
  • Week 5Load sentence-transformers/all-MiniLM-L6-v2 via HuggingFace. Generate embeddings for ingested text chunks (512-token windows with 50-token overlap).
  • Week 5Setup Elasticsearch. Index embeddings using dense_vector field + KNN search. Store alongside metadata (source, type, date).
  • Week 6Build cosine similarity scorer. Define thresholds: >0.90 = high risk, 0.75–0.89 = medium, 0.60–0.74 = low, <0.60 = clear.
  • Week 6Implement copyright text detection: chunk input → embed → KNN search → rank results by score → return top 10 matches with source metadata.
  • Week 7Build code license detector: use licensee + AST-level fingerprinting for Python/JS/Java. Map license types (MIT, GPL, Apache, proprietary).
  • Week 8Implement image similarity using CLIP (openai/clip-vit-base-patch32). Build perceptual hash fallback with imagehash library.
04
Patent Detection & Database Integration
Weeks 9–11 · USPTO/EPO/WIPO queries, patent matching
  • Week 9Integrate USPTO Open Data API. Build PatentSearchService: extract noun-phrase claims from input using spaCy → query USPTO full-text search API.
  • Week 9Integrate EPO Open Patent Services (OPS) API. Handle OAuth 2.0 client credentials flow. Parse XML responses to structured patent objects.
  • Week 10Integrate WIPO PATENTSCOPE API. Implement rate limiting + exponential backoff. Cache results in Redis (TTL: 24h) to reduce API quota usage.
  • Week 10Build patent relevance ranker: embed patent abstract → compute similarity with input embedding → rank candidates → return top 5 with claim excerpts.
  • Week 11Build RoyaltyFreeRecommendationEngine: for each flagged asset type, query Free Music Archive API / Unsplash API / SPDX license DB for legal alternatives.
05
Report Generator & Frontend Dashboard
Weeks 12–14 · Risk reports, UI dashboard, PDF export
  • Week 12Build RiskReportGenerator: aggregate all module results into a structured report object. Assign overall risk score (weighted average across modules).
  • Week 12Add jurisdiction-awareness: map detected IP to jurisdiction (US/EU/Global) based on database source. Display applicable law references (e.g., 17 U.S.C. § 106).
  • Week 13Build PDF export with WeasyPrint. Template via Jinja2: cover page, risk summary table, match details, recommendations, disclaimer.
  • Week 13Build React dashboard: project list, analysis history, risk badge system. Use Recharts for score visualizations. Build report viewer with highlighted match excerpts.
  • Week 14Build notification system: email via SendGrid when analysis completes. In-app notification centre. Optional Slack webhook integration.
06
Testing, Security & Deployment
Weeks 15–16 · QA, hardening, cloud deploy
  • Week 15Write pytest integration tests for all API routes. Write vitest + React Testing Library tests for all major components. Target ≥70% coverage.
  • Week 15Security hardening: add rate limiting (SlowAPI), input sanitization, CORS config, CSP headers. Run OWASP ZAP scan. Fix findings.
  • Week 16Dockerize all services. Write docker-compose.prod.yml. Deploy backend to Render or Railway. Deploy frontend to Vercel. Setup environment secrets via .env vault.
  • Week 16Setup Sentry for error tracking (backend + frontend). Add basic analytics (page views, job counts). Write final API docs via FastAPI's auto-generated OpenAPI spec.
Core user stories that define what LexaTrace must do, broken by persona.
Content Creator
As a YouTuber, I want to upload my video script and background music file before publishing, so I know if any content will trigger a copyright strike.
Acceptance criteria: System accepts TXT/DOCX and MP3/WAV. Returns similarity score with matched sources. Flags at-risk segments with timestamps or line numbers. Suggests royalty-free alternatives from FMA.
Software Developer
As a developer, I want to upload a ZIP of my codebase and get a license compatibility report, so I can ensure my open-source release is legally clean.
Acceptance criteria: System extracts all .py/.js/.java files. Detects license headers and fingerprints. Identifies GPL-contaminated dependencies. Outputs a SPDX-format compatibility matrix.
Startup Founder
As a startup founder, I want to describe my product concept and check it against global patent databases, so I can file my patent application with confidence.
Acceptance criteria: Input text (500–5000 chars). System queries USPTO, EPO, WIPO. Returns top 5 similar patents with claim excerpts and similarity score. Jurisdiction column on results.
Researcher
As an academic researcher, I want to check my paper draft for plagiarism and patent conflicts before submission, so I avoid IP disputes post-publication.
Acceptance criteria: Accepts PDF up to 15MB. Runs both copyright (text) and patent detection in parallel. Returns combined report with separate section for each. Export to PDF.
Legal Team
As a legal associate, I want to batch-analyze multiple client manuscripts via API, so I can integrate IP screening into our due diligence workflow.
Acceptance criteria: REST API with API-key auth. POST /analyze accepts file + metadata. GET /jobs/{id} returns async status + result. Webhook callback on completion. Rate limit: 100 req/hour per key.

External APIs LexaTrace must integrate with. Each requires registration and credentials management via environment variables.
USPTO Open Data API
Full-text search of US patents and patent applications. Free to use with registration. Returns patent metadata, claims, and abstracts in JSON.
https://developer.uspto.gov/api-catalog
EPO Open Patent Services (OPS)
Access to European Patent Office data. OAuth 2.0 client credentials. Supports full-text search, bibliographic data, and family lookups. Free tier: 4GB/month bandwidth.
https://developers.epo.org
WIPO PATENTSCOPE
International PCT application search. REST API with API key. Covers 100M+ patent documents across 70+ patent offices. Rate limit: 500 req/hour on free plan.
https://patentscope.wipo.int/search/en/help/apihelp.jsf
Free Music Archive (FMA) API
Discovery API for royalty-free music under Creative Commons. Used by the Royalty-Free Recommendation Engine to suggest music alternatives when audio IP conflicts are detected.
https://freemusicarchive.org/api
Unsplash API
Source of royalty-free images for recommendation engine. When image IP conflict is detected, suggest visually similar free alternatives. 50 req/hour on free tier.
https://unsplash.com/developers
SendGrid Email API
Transactional email for analysis completion notifications and report delivery. Free tier: 100 emails/day. Required env var: SENDGRID_API_KEY.
https://docs.sendgrid.com/api-reference

Known risks and mitigation strategies. Address high-severity items before starting the affected phase.
RiskSeverityMitigation
USPTO/EPO/WIPO API rate limits block patent search in production High Implement Redis caching (TTL 24h), exponential backoff, and a job queue to serialize requests. Consider a local patent dataset snapshot for demo.
BERT model inference too slow for real-time response High Use async Celery tasks — never block HTTP. Use MiniLM (80x smaller than BERT-large). Pre-compute & cache embeddings for known datasets. Quantize model with ONNX Runtime.
Elasticsearch memory usage exceeds free-tier cloud limits Medium Run single-node Elasticsearch in Docker with heap capped at 512MB for dev. For production, consider Elastic Cloud free tier or self-host on a VPS.
Image IP detection accuracy too low for reliable results Medium Set conservative thresholds and label low-confidence results explicitly. Use perceptual hash as a fast pre-filter before CLIP. Clearly communicate confidence levels in the UI.
Legal disclaimer requirements — tool output misinterpreted as legal advice High Add a prominent disclaimer on every report: "This is an automated similarity check, not legal advice. Consult a qualified IP attorney for legal decisions." Include in PDF export header and UI.
Large file uploads (PDFs, code ZIPs) timing out Medium Enforce 15MB upload limit. Use chunked streaming upload. Process entirely async via Celery. Return 202 Accepted immediately and poll via /jobs/{id} endpoint.
Team GPU access limited for model fine-tuning Low Use pre-trained models without fine-tuning for the IDP scope. Google Colab Pro (free tier) is sufficient for any needed experiments. Fine-tuning is a future enhancement.
